Frontend

Front end is the front part of the website this means it contains all related to what then user sees, Such as bottoms, titles, color, fonts, pictures, etc.

Backend

backend is related to all what the user can’t see such as data bases, connections to a server by the web page.

End User

The end user not necessarily is the one paying for the software. The end user is the one who will use the software directly. Take as example a restaurant, the restaurant can buy a software to administrate the orders, there the end user is the waiter using the software.

The Client

The client is the one paying for the software to be developed. Take as an example the previous restaurant. The client is the resultant because it is paying for the software to be developed and later on used by the restaurant´s employees.

CDN ARCHITECTURE

CDN stands for, Content delivery networks. CDN was born with the objective to solve the problem of internet latency, which is the delay from you ask the server the info, to the moment it appears on your screen. CDN is based on accessing several servers all around the world, so the delay time decreases by improving rendering and performance, also CDN creates a preloaded version of a page, from the previous user, so the next one gets a smaller latency.
